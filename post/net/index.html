.NET:

  • structure:
    • runtime component
    • languages
    • compilers
  • platforms:
    • .NET Core
    • .NET Framework
    • Xamarin
  • it provides definitions for a set of Base Class Libraries(BCL) APIs to be implemented by all the platforms conforming to the standard

.NET Core:

  • components:
    • CoreCLR(Common Language Runtime):
      • open source refactoring of the original CLR
      • manage usage and references to objects
      • communication and interactions of objects written in the different languages supported
      • garbage collection
      • components:
        • garbage collector
        • Just In Time(JIT) compiler
        • Native interop:
          • allow managed(.NET constructs) and unmanaged(c/c++ constructs) to coexist and interoperate
        • Base.NET types(!??!?!)
    • CoreFX:
      • a suite of foundational libraries:
        • primitive datatypes
        • filesystems
        • application composition types
        • consoles
        • basic utilities
    • Core SDK:
      • a set of tools to help the build of applications
      • components:
        • cli tools and compilers
        • application host(!?!?!?)
        • application launcher(?!!?)
  • v1.0 supported C# and F#, v2.0 added VB.NET
  • good at:
    • cross-platform
    • ease of deployment:
      • side-by-side versioning
      • easy to build in a container
    • focus on scalability and performance:
      • built with microservice paradigm in mind
      • easy to deploy to cloud
  • limitations:
    • lack of standard technologies:
      • WPF, WCF
    • relatively new, so some third-party libraries are not ported(or will never be)
