introduction in tcp/ip networking:

  • what is a network?:
    • multiple computers connected together
    • eases the process of moving data from a computer to another
  • what is a networking model?
    • big vendors each had their own communication protocols, these propietary protocols would work for their devices but not combined with other devices
    • ISO(international organization for standardization) first tried to standardize a less complex, vendor-neutral network model, which was OSI(open systems interconnection)
    • key terms:
      • adjacent-layer interaction(communication between two layers on the same computer)
      • same-layer interaction(communication at the same layer between two computers)
      • de-encapsulation: remove and parse header and trailer metadata
      • encapsulation: generate and add header and trailer metadata
      • frame(PDU at data-link)
      • packet(PDU at network layer)
      • segment(PDU at transport layer)
      • protocol data unit(PDU):
        • a generic way to refer to messages(segments, packets, frames)
        • data unit that will be sent to the lower layer
        • it contains:
          • current layer header
          • higher layer data unit
  • what is tcp/ip?
    • a networking model:
      • the most popular one
      • OSI is used only by its terminology so is it the only one?
    • created by the US DOD and volunteers
    • broken into layers:
      • physical:
        • how to transmit bits over each individual link
        • functions related to physical transmision of the data
        • protocols and rules that control the use of the physical media
      • data link: sending data over one type of physical link(one protocol for ethernet lan and another for wireless lan)
      • network:
        • deliver data over the entire path from the original sending computer to the final destination computer
        • popular protocols are IP(internet protocol)
      • transport:
        • popular protocols are TCP(transport control protocol) and UDP(user datagram protocol)
      • application: an interface between software running on a computer and the network itself

fundamentals of ethernet lans:

  • an overview of lans:

    • what is a lan?:
      • a network of interconnected networks and/or computers locally situated
      • types:
        • ethernet(cable linked)
        • wireless(no cable used)
    • what is a WAN?
      • a network of interconnected networks and/or computers situated far away
    • soho lan:
      • what are the components and function of a SOHO LAN?(small office/home office):
        • an ethernet switch
        • ethernet cables
        • an router
        • an wi-fi ap
        • the switch, the router and the ap are compressed in the same device
    • enterprise lan:
      • how does an enterprise lan differ from a soho lan?:
        • they are structurally the same, but the enterprise is used on a larger scale
        • SOHO usually support up to 20 devices, enterprise can support thousands
        • the switch, the router and the ap are each their own device, much more powerfull than the small parts of an SOHO router
    • the variety of physical layer standards:
      • what is a fiber cable?:
        • data is sent through fiberglass strands inside the cable, the data is encoded as light signals
      • what is a standard?:
        • rules for governing the transmission of data
        • some standards control HOW to send data over a cable and the SPEED
        • others define how ethernet devices should behave to be part of an ethernet LAN
      • enumerate a few standards:
        • the T stands for UTP
        • the X stands for fiber
