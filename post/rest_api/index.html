rest api:

  • write a bit of client code before design and implementation(readable code from a api, means the api is well written), it will force you to keep the client’s perspective and will give you a starting point
  • makes easy things easy and hard things possible !!!!!
  • take into account the use-cases and the public of you api
  • never start implementation before design
  • watch out for leakage of implementation details into the api, interfaces outlive implementations
  • ask for constant feedback, try dogfooding
  • name consistently
  • name should be a bit specific, it is hard to reclaim api real-estate
  • hard to misuse, sane defaults(false by default and enabled by the user)
  • property-based instead of providing all the params in the constructor

fundamentals of api design

  1. what is api design? 1.1. what is an api?:
  • a web api is a web interface for software
  • private/public api is not about how it is exposed, but to whom

1.2. why api design matters:

  • well-designed apis are easy to use
  • badly-designed apis can be misused or underused

1.3. the elements of api design:

  • be aware of how easy-to-understand and how easy-to-use is the interface
  • the api should be secure:
    • not export sensitive information
  • what constraints are
  • how will the api be used, and by whom
  • how it is build
  • how will the api evolve
  • design the api in collaboration with other api designers, to ensure all the organizations apis have the same feel and look
  1. designing an api for its users 2.1 the right perspective:
  • always focus on the user perspective 2.2 designing software’s interface:
  • an api is a control panel made to interact with programmatically
  • the api is the ui of the implementing programmer 2.3 identifying an api’s goals:
  • questions to ask yourself:
    • who can use the api?
    • what can they do?
    • how they do it?
    • what they need to do it?
    • what they get in return?
  • steps:
    1. identify whats and hows:
      • what?
      • how?
    2. identify inputs and outputs:
      • inputs: what is needed
      • outputs: what is returned(expected)
