  • What is an internetwork:
    • two or more networks connected via a router and configured with a logical network addressing scheme with a protocol such as ip or ipv6
  • network segmentation:
    • use switches to segment each sw interface into a separate collision domain
    • hubs connect network segments
    • routers break up a broadcast domain
  • network broadcast:
    • every device on the network must read and process the broadcast, unless its a router
  • collision domain:
    • a device sends a packet out on a network segment and every other device is forced to pay attention no matter what
    • if two devices try to send packets at the same time, a collision occurs and they need to retransmit the packet sequentially
  • devices:
    • routers:
      • break up broadcast domains
      • break up collision domains
      • don’t forward broadcasts by default
      • can filter the net based on layer 3(network layer) ip address(logical addressing)
      • packet switching
      • packet filtering:
        • via access lists
      • internetwork communication:
        • connect two or more networks together
      • path selection:
        • uses a routing table to map networks to paths
      • L3 switches
      • provide connection between virtual lans(vlans)
      • can provide quality of service(QOS) for specific types of network traffic
      • it only cares where a network is located and how to get to it
    • switches:
      • forward and filter frames
      • L2 switches
      • optimize LAN performance
      • does not break up domains by default
      • switch frames from a port to another withing a switched network
      • each port is a different collision domain
      • in a switched network, you still need a router to provide communication between VLANS
      • it only cares about uniquelly identifying a lan device
    • wlan device(wireless nic)
    • access points:
      • extend a collision domain
      • they are in their own broadcast domain(vlan)
    • firewalls:
      • controll incoming and outgoing net traffic
      • they are usually an ips(intrusion protection system)
  • common lan traffic congestion reasons:
    • too many hosts in a collision or broadcast domain
    • broadcast storms
    • too much multicast traffic
    • low bandwidth
    • adding hubs for connectivity to the network
    • a bunch of arp broadcasts

Communication modes:

  • simplex: one-way communication(say something and not get a reply)
  • half-duplex: two-way communication(receiver has to wait for the sender to finish sending the message, then it can send a response back)
  • full-duplex: two-way communication(devices receive and transmit at the same time)

Communication types:

  • connection-oriented:
    • caracteristics:
      • virtual circuit(three-way handshake)
      • uses sequencing
      • uses acknowledgments
      • uses flow control
    • steps:
      • first establish a connection-oriented session with a “three-way handshake”:
        • segment from sender to receiver to request for synchronization(SYN)
        • segment from receiver to sender to acknowledge(ACK) the request and establish connection parameters
        • segment from sender to receiver to acknowledge(ACK) that the connection agreement has been accepted and the actual connection has been established
      • transfer data
      • termination call that tears down the virtual circuit

Flow control:

(provide a way for the receiver to control the amount of data sent by the sender)

  • the segments delivered are acknowledged upon reception
  • any segments not acknowledged are retransmitted
  • segments are sequenced back into their proper order upon arrival at their destination
  • when a receiving buffer is full, sender gets a “not ready” message that signals to stop sending segment traffic to its overwhelmed receiver, and “ready” when the buffer can manage new segments

Windowing:

  • the quantity of data segments, in bytes, that the transmitting machine is allowed to send without receiving an acknowledgment

Acknowledgments:

  • the sender sends a segment(sets a timeout value) and waits for an acknowledgment, if an acknowledgment is received, send the next segment, if the timeout values is passed, resend the timeouted segment

Routing:

  • when a packet is received on an interface, the destination ip is checked
  • if the destination is not that router, the ip is checked in the routing table
  • if the ip is not found in the routing table, drop the packet
  • if the ip is found in the routing table, send the packet to that interface to be framed and sent out
  • keywords:
    • data packet:
      • transport data
    • route update packet:
      • used to build and update routing tables
      • update neighboring routers about the networks connected to all routers:
        • RIPv2
        • EIGRP
        • OSPF
    • network address:
      • protocol-specific network address
    • interface:
      • exit interface a packet will take when destined for a specific network
    • metric:
      • distance to the remote network:
        • different protocol use different metrics:
          • hop count(RIP)
          • bandwidth
          • delay of the line
          • tick count(1/18 of a second)

OSI Model:

(Open Systems Interconection reference model)

  • main goal was interoperability between network devices and software
  • structure: (upper layers;how apps within end stations communicate with each other and with the user)
    • application(7):
      • file, print, message, database, and application services
      • provides a user interface
    • presentation(6)
      • protocols that define how standard data should be formatted
      • data encryption, compression, and translation services
      • presents data
      • handles processing such as encryption
    • session(5):
      • set up, manage, and dismantle sessions
      • dialog control
      • keeps different applications data separate (lower layers; end to end data transmission)
    • transport(4):
      • segment and reassemble data into a single data stream
      • end-to-end connection
      • provides reliable or unreliable delivery
      • performs error correction before retransmit
    • network(3):
      • routing
      • provides logical addressing, which routers use for path determination
      • transports traffic between devices that aren’t locally attached
    • data link(2):
      • framing
      • combines packets into bytes and bytes into frames
      • provides access to media using MAC address
      • performs error detection not correction
      • flow control
      • network topology
    • physical(1):
      • physical topology
      • moves bits between devices
      • specifies voltage, wire speed, and pinout cables
