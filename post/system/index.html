Reasons why RHEL migrated from Upstart init daemon(RHEL 6) to systemd:

  • faster boot times because targets units startup can be paralelized(systemd-analyze time/systemd-analyze blame)
  • only needed units are started, others are started on demand
  • units use cgroups(linux kernel native) to group, control( ensure all processes of the service are terminated upon service stop) and limit processes started by a service(systemd-cgls)
  • a central logging system(journalctl):
    • for log disk persistence: mkdir /var/log/journal chgrp systemd-journal /var/log/journal chmod 2775 /var/log/journal systemctl restart systemd-journald.service

Location

  • default files are in /usr/lib/systemd/system, don’t overwrite this, they are writen upon software updates.
  • custom files are in /etc/systemd/system.

Unit Types(systemctl list-units):

  • Target -> a set of units
  • Service -> a service, daemon
  • Socket -> an listening IPC or network socket that activates a service when data is received(similar to xinetd)
  • Device -> drive, partition
  • Mount -> mount point controlled by systemd
  • Automount -> automount point controlled by systemd
  • Swap -> swap partition to be activated by systemd
  • Path -> monitored path that activates a service when changed
  • Timer -> activate a service when the timer elapses
  • Snapshot -> snapshot of the systemd run-time state
  • Slice -> group of system resources(CPU, memory, etc) assignable to an unit via the cgroup interface
  • Scope -> a unit for organizing and managing resource utilization of a set of system processes
