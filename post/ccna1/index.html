Theory:

  • introduction in tcp/ip networking:
    • what is a network?:
      • multiple computers connected together
      • eases the process of moving data from a computer to another
    • what is a networking model?
      • big vendors each had their own communication protocols, these propietary protocols would work for their devices but not combined with other devices
      • ISO(international organization for standardization) first tried to standardize a less complex, vendor-neutral network model, which was OSI(open systems interconnection)
      • key terms:
        • adjacent-layer interaction(communication between two layers on the same computer)
        • same-layer interaction(communication at the same layer between two computers)
        • de-encapsulation: remove and parse header and trailer metadata
        • encapsulation: generate and add header and trailer metadata
        • frame(PDU at data-link)
        • packet(PDU at network layer)
        • segment(PDU at transport layer)
        • protocol data unit(PDU):
          • a generic way to refer to messages(segments, packets, frames)
          • data unit that will be sent to the lower layer
          • it contains:
            • current layer header
            • higher layer data unit
    • what is tcp/ip?
      • a networking model:
        • the most popular one
        • OSI is used only by its terminology so is it the only one?
      • created by the US DOD and volunteers
      • broken into layers:
        • physical:
          • how to transmit bits over each individual link
          • functions related to physical transmision of the data
          • protocols and rules that control the use of the physical media
        • data link: sending data over one type of physical link(one protocol for ethernet lan and another for wireless lan)
        • network:
          • deliver data over the entire path from the original sending computer to the final destination computer
          • popular protocols are IP(internet protocol)
        • transport:
          • popular protocols are TCP(transport control protocol) and UDP(user datagram protocol)
        • application: an interface between software running on a computer and the network itself
  • fundamental networking caracteristics:
    • fault tolerance:
      • limit the amount of devices affected when there are problems on the network
      • provide redundancy; multiple paths to the same destination
      • provide a quick way to regain connectivity after network problems
    • scalability:
      • add new users/devices to a network without disturbing current connections/services
    • quality of service(qos):
      • certain types of data require higher priority in a network(video/phone call)
      • qos manages congestion in a network:
        • congestion:
          • when required transfer requirements are higher than provided
          • intermediary devices hold packets in memory(local cache) until the congestion ends
    • security:
      • main concerns:
        • data access
        • network infrastructure
      • main caracteristics of a secure network:
        • availability: the messages arrive in a timely manner
        • integrity: the messages have not been changed in transport
        • confidentiality: the messages have not been read in transport
  • rules of communication:
    • sender

    • formatting
    • speed
    • timing:
      • flow control: control the size and speed of the transfer
      • response timeout: how long to wait for a message and what to do when the time has run out
      • access method
    • delivery:

    • confirmation or acknowledgment

  • data-link layer:
    • sublayers:
      • LLC(logical link control; 802.2):
        • communicates with upper layer and lower device hardware
        • identifies and decides which protocol to use for media transfer
      • MAC(media access control):
        • data encapsulation and accessing the media
        • hardware implementation of protocols from data-link
        • controls the hardware, the transmision of data
        • protocols:
          • 802.3 Ethernet(wired transmision)
          • 802.11 WLAN(wireless transmission)
          • 802.15 WPAN(bluetooth, RFID, etc)
        • data encapsulation through:
          • frame delimiting: delimiters to identify fields within a frame
          • addressing: source and destination
          • error detection: the trailer that is used to detect transmision errors(FSC: frame check sequence)
    • topologies:
      • LAN:
        • star: multiple devices connected to a switch
        • extended star: multiple switches, each with a multitude of devices connected to it(the switches are also connected)
        • bus
        • ring
      • WAN:
        • point-to-point: permanent link between two endpoints
        • hub and spoke:
          • a central device makes the connection of different disparate branches(each branch is connected to the central device, but not between themselves)
        • mesh: every device is connected to every other device
    • types of communications:
      • simplex: A transmits and B receives, ALWAYS
      • half-duplex: when one transmits the other can only receive
      • full-duplex: both can transmit and receive at the same time
    • access control methods:
      • contention-based:
        • half-duplex
        • devices compete for the use of the network
        • only one can send at a time
        • these types of environments do not scale well under heavy media use
        • types:
          • CSMA/CD(carrier sense multiple access with collision detection) on legacy bus topology ethernet LAN:
            • the device checks the wire to see if data is currently sent on it, if so, it awaits a clear time to send its data
            • if two devices send data at the same time, both will detect the collision and reset the frames
          • CSMA/CA(carrier sense multiple access with collision avoidance) on wireless LAN:
            • the device that transmits the data will also send how much time will need to transmit it fully, other devices will wait their turn based on the time duration received
      • controlled access:
        • each node has a set time to send data on the network
        • examples:
          • legacy token ring
          • legacy ARCNET
        • today ethernet operates in full-duplex, so this is not used
    • protocols:
      • depends on the technology used to implement the topology
      • the technology used depends on:
        • size of the network
        • number of hosts
        • geographic scope
        • services to be provided on the network
      • examples:
        • Ethernet
        • 802.11 Wireless
        • PPP
        • HDLC(high-level data link control)
        • Frame Relay
    • the frame:
      • size of minimum 64 bytes, and maximum 1518 bytes(header + data + trailer - preamble)
      • anything less or more than the limits, the frame is a “collision fragment” and is rejected by the device
      • structure:
        • preamble(7 bytes) and frame start(1 byte):
          • signals that a new frame will be sent
        • header:
          • addressing:
            • destination(6 bytes):
              • unicast
              • multicast
              • broadcast
            • source(6 bytes)
          • type(2 bytes; L3 protocol identificator)::
            • IPv4: 0x800
            • IPv6: 0x86DD
            • ARP: 0x806
          • control(qos flow control)
        • pdu(45-1500 bytes; network packet)
        • trailer(4 bytes):
          • error detection: CRC(ciclic redundancy check; a goddamn checksum)
          • frame stop

    • IP(internet protocol)
      • caracteristics:
        • best-effort: source does not know if the destination is online or if it received the packet
        • connectionless
        • media independent
      • types:
        • IPv4:
          • structure:
            • version(4 bits):
              • 0100 is IPv4
            • used for validation:
              • internet header length(4 bits): length of the entire header
              • total length(2 byte): header + data size
              • header checksum(2 byte): recalculated after every TTL decrement
            • DS(1 byte; differentiated services; formerly called ToS: type of service):
              • DSCP(6 bits; differentiated services code point)
              • ECN(2 bits; explicit congestion notification)
            • used for fragmentation:
              • identification(2 byte)
              • flag
              • fragment
            • TTL(1 byte; time to live):
              • set by source
              • decremented by every router(hop)
              • if it reaches 0, the router sends a ICMP: Time Exceeded to source
              • this means the decrementing router needs to also recalculate the header checksum
            • protocol(1 byte; transport layer protocol):
              • ICMP: 1
              • TCP: 6
              • UDP: 17
            • source(4 byte) and destination(4 byte) address
          • default gateway is set manually or received by DHCP
          • data-link sends the MTU(maximum transmission unit) to network:
            • if a router needs to send a packet from a network with higher MTU to a network with lower MTU, the router splits(fragmentation) the router in smaller packets that follow the MTU rule, this can cause latency
        • IPv6:
          • structure(length fixed at 40 bytes):
            • version(4 bits):
              • 0110 for IPv6
            • traffic class(1 byte): same as IPv4 DS
            • flow label(20 bits)
            • payload length(2 byte): length of payload
            • next header(1 byte): sames as IPv4 Protocol
            • hop limit(1 byte): same as IPv4 TTL
            • source(16 byte; 128 bit) and destination(16 byte; 128 bit) address
            • it can also include EH(extension headers) between header and payload: fragmentation, security, mobility, etc
          • improvements over IPv4:
            • increases available address space: 128 bit address length vs IPv4’s 32 bit
            • improves packet handling: fewer fields than IPv4
            • eliminates the need for NAT: the high number of addresses makes NAt useless
          • default gateway is set manually or adressed by the router to the whole network
          • a router can not fragment packets
      • address types:
        • unicast:
          • GUA(global unicast address):
            • for communication with external hosts
            • structure:
              • global routing prefix: assigned by the provider
              • subnet ID: link prefix
              • interface ID: host unique address
            • EUI-64(interface ID GUA generation algorithm):
              • 24bit MAC address OUI(7th bit is reversed)
              • 16-bit fffe
              • 24-bit device identifier from MAC
          • LLA(link-local address):
            • for communication inside the same subnet
            • windows uses the same algorithm for GUA and LLA generation
            • routers create a LLA when a GUA is created
            • in the range of fe80::/10 - febf::/10
          • loopback: ::1/128
        • multicast:
          • well-known:
            • ff02::1 all nodes multicast group
            • ff02::2 all routers multicast group
          • solicited node:
            • sent to a hardcoded multicast address that the ethernet frame will receive unconditionally
        • anycast:
          • an address assigned to multiple devices
          • the connection from a source and an anycast address is an unicast connection between the source and the nearest device with the assigned anycast address
    • MAC address descovery:
      • IPv4 ARP:
        • a host mantains a local ARP IP to MAC table(ARP cache)
        • a broadcast ARP request is sent and only the destination ip will respond with an unicast, the broadcaster will add the IP-MAC mapping to its table:
          • request:
            • destination is FF-FF-FF-FF-FF-FF
            • type is 0x806(ARP)
          • reply:
            • destination is request-source-mac
            • source is reply-source-mac
            • type is 0x806(ARP)
          • if no device responds to an ARP request, the packet is dropped(no destination to send to)
        • table entries are removed at expiry time(on windows is between 15 and 45 s) or manually
      • IPv6 ICMPv6 ND(network discovery):
        • messages:
          • neighbor solicitation(ARP request on a multicast address)
          • neighbor advertisment(ARP reply on a unicast address)
          • router solicitation(router communicates the default gateway)
          • router advertisment(router communicates the default gateway):
        • they are sent regularlly, every 200 seconds
        • structure:
          • network prefix
          • prefix length: an ipv6 LLA, the source of the RA
          • default gateway address
          • dns addresses
          • domain name
        • methods:
          • slaac:
            • host uses RA response to generate its own GUA
          • slaac with stateless DHCPv6:
            • host uses RA response to generate its own GUA
            • uses router LLA as default gateway
            • communicate with DHCPv6 for other information
          • statefull DHCPv6:
            • uses router LLA as default gateway
            • talk with DHCPv6 to obtain all other information
              • redirect
    • addressing:
      • in a network:
        • network address(first ip in subnet)
        • broadcast address(last ip in subnet)
        • host address(all other ips)
      • from a host:
        • unicast: one-to-one
        • broadcast: one-to-all(all devices on the network)
        • multicast:
          • one-to-group
          • all devices that subscribed to a group
          • IPv4 reserves 224.0.0.0 - 239.255.255.255 as a multicast range
      • private address blocks:
        • 10.0.0.0/8: 10.0.0.0 - 10.255.255.255
        • 172.16.0.0/12: 172.16.0.0 - 172.31.255.255
        • 192.168.0.0/16: 192.168.0.0 - 192.168.255.255
      • loopback addresses:
        • 127.0.0.0/8: 127.0.0.1 - 127.255.255.255
      • link-local addresses:
        • 169.254.0.0/16: 169.254.0.1 - 169.254.254.254
        • used by windows when no DHCP is available

  • ICMP(internet controll message protocol):
    • messages:
      • ipv4:
        • host echo
        • host reply
        • detination unreachable:
          • net unreachable(0)
          • host unreachable(1)
          • protocol unreachable(2)
          • port unreachable(3)
        • time exceeded
      • ipv6:
        • host echo
        • host reply
        • destination unreachable:
          • no route to destination(0)
          • communication with the destination is administratively prohibited; firewall(1)
          • beyond scope of the source address(2)
          • address unreachable(3)
          • port unreachable(4)
        • router solicitation
        • router advertisement
        • neighbour solicitation:
          • happens in the process of aquiring a GUA
          • send a NS with the generated GUA as destination
          • if a device has already the GUA assigned, it will send a NA(neighbor advertisement)
          • if the source does not receive a NA in a set timelimit the GUA is his
        • neighbour advertisement:
          • reply to a NS
        • time exceeded

  • multiplexing:
    • the act of segmenting data in packets that can be interleaved with other packets, as to not clog the network with a single unending data stream from A to B
    • if a packet fails to reach the destination, only that packet will be resent, insted of restarting the whole data transfer process
  • bandwidth: total amount of data that can be transferred
  • digital bandwidth: the amount of transferred data per time
  • throughtput: the measure of usable data trasferred(with retries, timeouts and headers counted)
  • latency: the duration of travel(with delays) from A to B
