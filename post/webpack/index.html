webpack:

  • info:
    • highly flexible(and complex) asset bundler
  • concepts:
    • entry:
      • module entrypoints
      • dependency tree roots
    • output:
      • output paths of bundles for every module in entry
    • loader:
      • webpack can only handle js and json files, loaders help webpack interact with other types of files as if they where js modules, esentially extending the dependency graph to include other types of files
      • structure:
        • test: what files to match
        • use: what loader to use on matched files
    • plugins:
      • custom js code that extends the functionality of webpack to that of a task runner, potentially ANY computable task, because js is turing complete
    • mode:
      • enable optimizations based on environment
      • values:
        • development:
          • DefinePlugin:
            • define constants at compile time, that can be used in the scripts at runtime
            • “process.env.NODE_ENV” is set to “development”
        • production:
          • mangle names
          • enable some internal plugins:
            • TerserWebpackPlugin: js minification
            • NoEmitOnErrorsPlugin: don’t emit assets on error
            • ModuleConcatenationPlugin: load all modules in the same scope closure, which translates into a faster execution time
            • FlagIncludedChunksPlugin: add chunk ids, guarding the adding of the same chunk again
            • FlagDependencyUsagePlugin: find what imports module1 uses from module2, adding used imports to module2 exports object
        • none

cli(webpack-cli):

  • build: bundle once
  • checkconf: check config
  • watch: watch files in entry and bundles on change
  • init: similar to npm init

babel:

  • packages:
    • @babel/core: main lib
    • @babel/preset-env: lib that provides a list of presets for popular setups
    • babel-loader: lib to bridge the gap betweeb webpack and babel(makes babel available in webpack)
    • babel-preset-react-app: a babel preset used in create-react-app
  • info:
    • transpile new js syntax to old js syntax, to make compatible with as many browsers and setups as possible
    • uses shims and polyfils
    • you describe the amount of compatibility desired
  • locations:
    • package.json -> 'browserslist'
    • webpack.config.js -> module.rules[0]

react:

  • component-based UI library
  • builds a tree of components, and handles state changes to their virtual-dom

react-dom:

  • glue code that bridges the gap between react and the DOM
  • code that renders react component tree in the browser
