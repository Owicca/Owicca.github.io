basic concepts:

  • shareable vs unshareable:
    • shareable: stored on one system and used on other systems
    • unshareable: !shareable
  • variable vs static:
    • static: files that do not change without sysadmin intervention(libraries, binaries, documentation, etc)
    • variable: !static
  • static files can be stored on read-only media, and their backup schedules is different than variable files

matrix:

shareableunshareable
static/usr//etc/
/opt//boot/
variable/var/mail//var/run/
/var/spool/news//var/lock/

fhs:

  • /:
    • used for system maintenance: boot, recover, restore, repair
    • children:
      • /bin/:
        • binaries for use by all users
        • no subdirectories allowed
        • required children:
          • cat, chgrp, chmod, chow, cp, date, dd, df, dmesg, echo, false, hostname, kill, ln, login, ls, mkdir, mknod, more, mount, mv, ps, pwd, rm, rmdir, sed, sh, stty, su, sync, true, umount, uname, test, [
      • /boot/:
        • bootloader static files
        • kernel is stored here
        • stores everything required to boot the system
        • everything that is used before the kernel loads user-land apps
        • receives help from /sbin/ binaries
        • boot configuration is stored in /etc/
      • /dev/:
        • device files
      • /etc/:
        • host-specific configuration files
        • no binaries allowed, only non-executable static files
        • required children:
          • /etc/opt/:
            • configuration files for third-party applications
      • /home/:
        • user home directories
        • storage of user-specific settings, applications, files
      • /lib/:
        • shared libraries and kernel modules
        • repository of shared libraries or kernel modules used by /bin/ and /sbin/ apps
        • required children:
          • /lib/modules/:
            • for kernel modules
          • /lib/cpp/:
            • if a C preprocessor is installed
          • libc.so.*:
            • libc
          • ld*:
            • execution-time linker
      • /lib64/:
        • 64 bit variant of /lib/
        • required children:
          • same as in /lib/ except for /lib64/cpp
      • /media/:
        • removable media mount point
        • required children:
          • /media/floppy/:
            • floppy drive
          • /media/cdrom/:
            • cd-rom drive
          • /media/cdrecorder/:
            • cd writer
          • /media/zip/:
            • zip drive
      • /mnt/:
        • temporary mount point
        • this must NOT be used by installation programs
      • /opt/:
        • third-party application storage
      • /root/:
        • root user home directory
        • just a recommandation
      • /run/:
        • running processes data
        • contains data from the startup of the system
        • it is cleared at the start of the boot process
        • /var/run/ is used for legacy apps
        • pid files:
          • /run/<app_name>.pid: <ascii encoded pid as decimal>\n
      • /sbin/:
        • essential binaries, used mainly in recovery, restoring or repair
        • binaries used by administrators
        • required children:
          • shutdown
      • /srv/:
      • /tmp/:
        • temporary files: caches, download results, sockets, lock files
        • it is recommended to delete the contents on boot
  • /usr/:
    • shareable read-only data
    • required children:
      • /usr/bin/
      • /usr/lib/
      • /usr/lib64/
      • /usr/local/
      • /usr/sbin/
      • /usr/share/
    • children:
      • /usr/bin/:
        • executable applications
        • subdirectories are not allowed
      • /usr/include/:
        • header files for C lang
      • /usr/lib/:
        • shared libraries
        • required children:
          • /usr/lib/sendmail
      • /usr/lib64/
        • 64bit shared libraries
      • /usr/libexec/
      • /usr/local/:
        • tf is this?!?!
      • /usr/sbin/:
        • non-essential libraries only used by admins
      • /usr/share/:
        • read-only data files
        • required children:
          • /usr/share/man/:
            • manual location
            • man structure:
              • man1:
                • publicly accesible user commands
                • usefull to every user
              • man2:
                • system calls
                • only usefull to programmers
              • man3:
                • library calls
                • only usefull to programmers
              • man4:
                • special files
              • man5:
                • various data file formats:
                  • include files
                  • program output files
                  • system files
              • man6:
                • games
              • man7:
                • miscellaneous
              • man8:
                • system administration programs
          • /usr/share/misc/:
            • applications that use only one file
      • /usr/src/:
        • source code
  • /var/:
    • contains variable data
      • logs
      • temporary files
      • transient files
      • spool directories and files
    • required children:
      • /var/cache/:
        • application cache
        • applications should be able to recover if the contents are deleted
        • data is valid after system reboot or between multiple invocations of the same application
      • /var/lib/:
        • state data used by applications
        • data is valid after system reboot or between multiple invocations of the same application
        • required children:
          • /var/lib/misc/:
            • for state files that don’t need a subdirectory
      • /var/local/:
        • variable data for packages in /usr/
      • /var/lock/
      • /var/log/:
        • logging output
      • /var/opt/:
        • variable data for packages in /opt/
      • /var/run/:
        • used for historical reasons, the contents have moved to /run/
      • /var/spool/:
        • data which awaits some kind of later processing
        • contents are deleted after begin processed
      • /var/tmp/:
        • temp files preserved between system reboots
    • reserved children(should not be used by any application, reserved for historical purposes):
      • /var/backups/
      • /var/cron/
      • /var/msgs/
      • /var/preserve/

linux:

  • /:
    • if kernel is located here, it should be named vmlinux or vmlinuz
  • /dev/:
    • required children:
      • /dev/null:
        • all data written is discarded
        • read attempts return an EOF
      • /dev/zero:
        • all data written is discarded
        • read returns zero continously
      • /dev/tty:
        • the controlling terminal of a process
  • /proc/(virtual filesystem):
    • controlling system and processes
  • /sys/(virtual filesystem):
    • information about devices, drivers and some kernel features
  • /usr/include/
    • headers for C lang
  • /var/spool/cron/:
    • variable data for cron and at jobs
